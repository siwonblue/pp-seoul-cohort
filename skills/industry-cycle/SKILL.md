# 산업 사이클 분석 — 단일 산업 딥다이브

프프서울 1주차의 핵심 워크플로우입니다. 한 산업을 골라 사이클 위치를 가설로 잡고, 5개 지표로 검증하고, 반박 가능성을 명시하는 것이 목표입니다.

## 코호트 원칙 (반드시 준수)

- 종목 매수/매도 추천 금지. 산업/섹터 레벨에서 멈춥니다.
- 모든 결론은 **가설**로 표현. 근거 자료와 반박 가능성을 함께 남깁니다.
- AI는 결론 대신 내리는 도구가 아니라, 질문을 정리하고 빠진 근거를 찾는 보조 도구.
- 단기 매매보다 장기·가치 투자 관점의 리서치 습관.

## 절차

### 1) 산업 확정

사용자에게 묻습니다:

> 어떤 산업을 분석할까요? (예: 반도체, 자동차, 헬스케어, 2차전지, 금융, 에너지, 부동산, 미디어 광고, 항공 등)

답이 없으면 "반도체"로 시작 (1주차 데모 기본값).

### 2) 사이클 프레임 제시 (사용자 학습용)

다음을 짧게 설명:

| 단계 | 특징 |
|---|---|
| 초기 확장 (Early) | 수요 회복, 재고 정상화, 가격 안정, 마진 회복, 신규 capex 시작 |
| 중기 확장 (Mid) | 수요 강세, 재고 타이트, 가격 상승, 마진 피크 근처, capex 본격화 |
| 후기 확장 (Late) | 수요 둔화 조짐, 재고 빌드업, 가격 정점, 마진 압박, capex 과잉 신호 |
| 수축 (Contraction) | 수요 감소, 재고 과잉, 가격 약세, 마진 적자 위험, capex 동결/감축 |

### 3) 5개 지표 점검

각 지표마다 **현재 수치/추세 + 출처**를 정리. 모르는 부분은 WebSearch/WebFetch로 확인.

- (a) **수요**: 출하량/판매량/오더 백로그 추세. 최근 12개월.
- (b) **재고**: 메이커 재고/유통 재고/DOH(Days of Inventory).
- (c) **Capex**: 주요 기업 capex 가이던스, capacity utilization.
- (d) **가격·마진**: ASP, GP%, OP%. peer 평균과 비교.
- (e) **자본조달**: 회사채 발행, 금리, P/B, 산업 valuation multiple.

거시 컨텍스트 출처는 Federal Reserve (https://www.federalreserve.gov/feeds/press_all.xml), FRED, BOK, KOSIS 우선. 산업별 데이터는 회사 IR, KRX, DART, 또는 신뢰 가능한 매체 인용.

### 4) 결론을 가설 형태로

다음 마크다운 구조로 출력:

```markdown
## 가설
[산업명]은 현재 [단계] 국면에 있다.

## 근거 5개
1. 수요: ... — 출처: ...
2. 재고: ... — 출처: ...
3. Capex: ... — 출처: ...
4. 가격·마진: ... — 출처: ...
5. 자본조달: ... — 출처: ...

## 반박 가능성 3개
1. ... (만약 이게 사실이면 가설이 깨짐)
2. ...
3. ...

## 다음 주 입력값
- 2주차에 더 알아볼 기업 후보 3개: ...
- 추가로 추적할 지표 1-2개: ...
```

### 5) 출처 검증 안내

사실확인이 필요한 주장이 있으면 사용자에게 안내:

> 위 근거 중 출처가 불확실한 것은 `/journalism-core:source-verification` 또는 `/journalism-core:fact-check-workflow` 로 검증해 보세요. (claude-skills-journalism 플러그인 필요)

### 6) 노트 저장 제안

마지막에:

> 이 결과를 1주차 노트로 저장하려면 `/pp-seoul-cohort:weekly-note` 를 실행하세요.

## 안전 규칙

- 종목 단위 매수/매도/목표가 출력 금지.
- 출처 없는 수치 추측 금지. 모르면 "출처 미확인"으로 표기.
- 반박 가능성은 반드시 3개 이상.
- 1주차 학습 목적이므로, 결론보다 **근거를 모으는 절차** 자체를 사용자가 따라할 수 있게 단계를 보여주세요.
