# 1주차 리서치 노트 자동 생성

프프서울 1주차 마무리 단계. 진행된 분석을 한 페이지로 압축해 저장합니다.

## 코호트 원칙

- 종목 추천 금지.
- 결론은 가설 + 근거 + 반박 가능성 + 다음 주 입력값.
- 동일 형식 반복 → 4주 후 한 묶음으로 보기 쉬워짐.

## 절차

### 1) 컨텍스트 수집

직전 대화에서 다음을 찾아 발췌:

- `cycle-map` 결과 (산업 비교 표 + 딥다이브 추천)
- `industry-cycle` 결과 (단일 산업 가설 + 5근거 + 반박)
- 사용자가 추가로 메모한 내용
- `journalism-core` 의 fact-check 통과/실패 항목 (있다면)

없는 부분은 빈 자리로 두지 말고 사용자에게 짧게 묻습니다:

> 다음 항목이 비어있습니다. 채워주실 수 있을까요?
> - 분석한 산업 1개:
> - 결론 가설 한 줄:
> - 가장 강력한 근거 1개:

### 2) 저장 경로 결정

- 디렉터리: `~/Documents/pp-seoul-notes/week1/`
  - 존재하지 않으면 `mkdir -p`
- 파일명: `<산업-슬러그>-<YYYY-MM-DD>.md` (산업명은 한글 그대로, 공백은 하이픈)
- 예: `~/Documents/pp-seoul-notes/week1/반도체-2026-05-13.md`

Bash 한 줄로 디렉터리 보장:

```bash
mkdir -p "$HOME/Documents/pp-seoul-notes/week1"
```

Windows PowerShell 환경에선:
```bash
powershell.exe -NoProfile -Command "New-Item -ItemType Directory -Force -Path \"$HOME/Documents/pp-seoul-notes/week1\""
```

### 3) 노트 본문 (고정 템플릿)

Write 도구로 다음 형식 그대로 파일에 저장:

```markdown
# 1주차 리서치 노트 — [산업명]

작성일: YYYY-MM-DD
작성자: [사용자 이름 또는 비워둠]

## 가설
[산업명]은 현재 [사이클 단계] 국면에 있다.

## 근거 5개
1. 수요: ... — 출처: ...
2. 재고: ... — 출처: ...
3. Capex: ... — 출처: ...
4. 가격·마진: ... — 출처: ...
5. 자본조달: ... — 출처: ...

## 반박 가능성 3개
1. ...
2. ...
3. ...

## 사실확인 메모
- fact-check 통과: ...
- fact-check 보류: ...
- 출처 검증 필요: ...

(journalism-core 미사용 시 "검증 미실시"로 표기)

## 다음 주(2주차) 입력값
- 더 알아볼 기업 후보 3개: ...
- 추가로 추적할 지표 1-2개: ...
- 사이클 가설 재검증 시점: [날짜]

## 비고
- 사이클 맵에서의 상대 위치: ...
- 다른 후보 산업 대비 매력도: ...

---
*프프서울 코호트 — 종목 추천 아닌 리서치 노트입니다.*
```

### 4) 사용자 확인 + 저장 완료 안내

저장 후 다음 한 줄:

```
✅ 1주차 노트 저장 완료
   경로: <전체 경로>
   2주차 시작 전, 노트의 "다음 주 입력값"을 확인해 주세요.
```

### 5) 다음 단계 안내

> 2주차 스킬은 추후 같은 마켓플레이스 업데이트로 들어옵니다.
> `/plugin marketplace update pp-seoul-cohort` 한 번 돌리면 받습니다.

## 안전 규칙

- 비어있는 섹션을 임의로 채우지 마세요. 모르면 "(미입력)"으로 표기.
- 같은 산업·날짜에 노트가 이미 있으면 덮어쓰지 말고 `-v2.md`, `-v3.md` 식으로 증분 저장.
- 파일 경로는 사용자에게 항상 절대 경로로 보여주세요 (~/... 가 아니라 펼친 경로).
